Edward and his webdev projects

Hey there! I found my calling when, wanting to learn Chinese in a relevant way, I wrote a English-to-Chinese chatroom. Since then, I have focused on making useful, perhaps fun, apps, with the addition of modern technologies like React and the frontend ecosystem.

Dividewise

Dividewise is my clone of the bill-splitting website Splitwise. On the backend, I wrote it using the Ruby on Rails MVC framework and PostgreSQL as the database. The frontend was a work of React, Redux, Styled Components, CSS and HTML. More recently, I've stuck in a google authentication inside the Rails controller with a google login page for OAuth! I especially enjoyed adding tons of authenticating logic to make sure only people with the proper access could see and change bills because I believe that is important when it comes to pecuniary concerns.

  • Split a bill!

Typing Fantasy

Typing Fantasy is a typing game that uses Javascript, Canvas, CSS and the principles of OOP. I fetched google translation APIs to voice the words that appear above enemies which you can type to attack. Words per minute is tracked as well as a player's health. Additionally the game logic was bifurcated into a Chinese and English version and there is logic to load only the dictionary needed (to save a player's bandwidth!). High scores are tracked in a in a json file to fit the interchange use-case. Keeping track of rendered frames and "layers" of what was on top of what was a real joy for me in this project. Be sure to try it out!

  • Show off your typing skills!

Bone Throw

This dog walk tracking site is a fullstack application that leverages xyz technologies. One of the main features was of keeping track of sensitive info like a dogs weight which we chose to store in a browsers local storage rather than in a database to keept hat info private

Bone Throw is a showcase of in-browser functionality like Local Storage to keep track of sensitive info like a dog's weight on users' browsers rather than in a traditional database. Furthermore I made a Python Django API that fetched from a Yahoo stocks API and transformed that stock data into a continuous wave with another physics API before then sending it off to Bone Throw for the user to see as a beautifully rendered Charts.js data chart! Data 'streams' like that of a user-input walk duration were leveraged by the MongoDB database and shown off to the world to keep users engaged and informed of a walk-duration recommendation.
